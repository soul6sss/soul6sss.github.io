第七章 软件测试

7.1软件测试概述

  1. 软件测试是发现软件中错误和缺陷的主要手段

  2. 软件测试的目的是检测软件故障，以下是软件测试的原则

    1. 完全测试不可能

    2. 测试中有风险

    3. 软件测试只能表示缺陷的存在

7.2软件测试模型

软件测试模型是指软件测试的全部过程活动和任务框架

问题

  1. 测试的时间

  2. 测试的步骤

  3. 如何对测试进行计划

  4. 不同阶段测试应该关注的对象

  5. 测试过程中应该考虑哪些问题

  6. 测试需要达到的目标

模型

  1. V模型：瀑布模型的变种

  2. W模型：由两个分别代表开发过程的V模型组成：优势在于测试活动可以并行进行

  3. H模型：强调测试的独立性和灵活性

测试用例和分类

  1. 编写测试用例

  2. 测试用例

  3. 测试用例场景

  4. 分类简述在P129页

软件测试方法

  1. 黑盒测试

    1. 等价划分法：划分若干子集，选出有代表的

    2. 边界值分析法：选取等价边界的测试用例

    3. 错误推测法：靠直觉和经验执行

    4. 因果图法：从自然语言书写的程序规格说明书中寻找因果关系，通过因果图产生验证表

    5. 决策表法：分析和表达多逻辑条件下执行不同操作的的情况的工具，把复杂逻辑和组合问题执行的情况的表述

    6. 场景法：包含基本流和备用流

  2. 白盒测试（成本较高）

    1. 代码检查法，检查代码的一致性

    2. 静态的结构分析法

    3. 程序插装技术

    4. 逻辑覆盖法：覆盖测试是百盒测试的重点

    5. 基本路径法

  3. 灰盒测试结合了上两个

单元测试：编写一个函数，检查其功能是否正常执行

集成测试：把多个单元组合起来进行测试，应该在单元测试之后{

  1. 非增量式集成测试：一步到位

  2. 增量式集成测试：

    1. 自顶向下：及早的发现问题，避免返工

    2. 自底向上：大大减少了装模块的开发

}

系统测试

  1. 功能测试

  2. 性能cs

  3. 安装测试

  4. 可用性测试

第九章 面向对象的软件工程方法

9.1面向对象的基本概念

  1. 面向对象：即按人们认识客观世界的系统思维方式，采用基于对象的概念建立模型，模拟客观世界分析，设计，实现软件的办法。通过面向对象的理念，计算机软件系统能与现实世界中的系统一一对应

  2. 对象，即现实世界中各种各样的实体。它可以指具体的事物，在面向对象概念中我们把对象的内部状态称之为属性，把运动规律叫做称作操作和服务

  3. 类：具有相似内部运动状态和运动规律的实体的抽象

  4. 消息：对象之间相互联系和相互作用的方式

  5. 类的特性

    1. 抽象

    2. 继承

    3. 封装

    4. 多态

    5. 重写

    6. 包

    7. 包的接口类

9.2软件工程方法的特征和优势

  1. 面向对象的软件工程方法的特征

    1. 将数据和操作封装在一起

    2. 将特征相似的对象抽象为类

    3. 对象之间通过发送信息来进行通信

    4. 将对象的私有信息封装起来

  2. 面向对象的软件工程方法的优势

    1. 符合习惯

    2. 稳定性好

    3. 可复用性好

    4. 可维护性好

  3. 面向对象的实施步骤：分析，域，设计，实现，测试
