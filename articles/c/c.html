解释

  1. #include ：引入对应库

  2. int main():程序的入口函数

3. 标准的输入输出

  1. printf("Hello\n"); 下面是常见的类型

  1. 输入

xxxxxxxxxx
int x;
scanf("%d",&x);

scanf需要取地址符

3️⃣ 多个输入输出

xxxxxxxxxx
int a, b;
scanf("%d %d", &a, &b);
printf("a=%d b=%d\n", a, b);

当你写 scanf("%d,%d", &a, &b); 时，你在格式字符串里放了一个逗号。

它的含义是：你告诉程序，用户输入的时候必须在两个数字之间也输入一个逗号。

如果你输入：10 20（用空格隔开）

scanf 成功读取了 10 赋值给 a。

接下来它在输入缓冲区里看到了一个空格，但你的代码要求它看到一个逗号。

由于匹配失败，scanf 会直接“罢工”退出，不再读取 b 的值。此时 b 是一个随机的垃圾值，相乘的结果自然也是乱码。

C 语言第 1–2 章易错点速查（只留关键）

一、printf / scanf（第 1 章高频错误）

1️⃣ printf 保留小数位

xxxxxxxxxx
printf("%.2f", x);

  • %.2f → 保留 2 位小数

  • 只影响输出，不改变变量本身

2️⃣ scanf 必须传地址

xxxxxxxxxx
scanf("%d", &a);   // 正确

❌ 错误：

xxxxxxxxxx
scanf("%d", a);

📌 唯一例外：数组名（以后学）

3️⃣ scanf 的格式 = 输入格式

xxxxxxxxxx
scanf("%d %d", &a, &b);   // 输入：3 4
scanf("%d,%d", &a, &b);   // 输入：3,4

📌 格式不匹配 → 输入失败

4️⃣ 字符串 vs 字符

xxxxxxxxxx
printf("%c", 'A');   // 字符
printf("%s", "A");   // 字符串

❌ 错误：

xxxxxxxxxx
printf('%s', "A");

二、数据类型与运算（第 2 章重灾区）

5️⃣ 整数除法一定会截断

xxxxxxxxxx
5 / 2      // 2

📌 只要两个操作数都是 int → 结果是 int

6️⃣ 想要小数，必须有浮点数

xxxxxxxxxx
(double)5 / 2
5 / 2.0

📌 推荐：

xxxxxxxxxx
(a + b) / 2.0

7️⃣ % 只能用于整数

xxxxxxxxxx
5 % 2      // ✅
5.0 % 2    // ❌

8️⃣ char 本质是整数（ASCII）

xxxxxxxxxx
char c = 'a';
printf("%d", c);   // 97

📌 字符 ≠ 字符串

9️⃣ = 和 == 不是一回事

xxxxxxxxxx
a = 5;     // 赋值
a == 5;    // 判断

📌 if (a = 5) → 逻辑错误但可能不报错

三、未定义行为（UB）——必须牢记

🔥 10️⃣ 同一表达式中多次修改同一变量 → UB

xxxxxxxxxx
a++ + ++a   // ❌ 未定义行为

原因：

  • 修改了 a 两次

  • 中间没有序列点

  • C 标准不规定执行顺序

📌 结果不确定，不能猜

11️⃣ 正确写法：拆开

xxxxxxxxxx
x = a++;
y = ++a;

12️⃣ Java vs C（一句话记忆）

C 不保证顺序 → UB Java 规定左到右 → 有结果，但仍不推荐

四、强烈建议你记住的 5 句口诀

1️⃣ scanf 读变量 → 一定要 & 2️⃣ 两个 int 相除 → 没小数 3️⃣ % 不能算小数 4️⃣ char 是数字 5️⃣ 一个表达式里别动同一个变量两次

第 6 章：指针（C 语言 · 核心中的核心）

学完这一章，你会突然发现： 数组、函数、字符串、结构体、malloc —— 全都连起来了。

一、指针到底是什么？（一句话定生死）

指针 = 存地址的变量

不是“高级东西”，不是“玄学”。

1️⃣ 普通变量 vs 指针变量

xxxxxxxxxx
int a = 10;
int *p = &a;

内存视角（非常重要）

xxxxxxxxxx
地址        内容
0x1000      10        ← a
0x2000      0x1000    ← p

二、& 和 * 到底是什么意思（重点）

1️⃣ &：取地址

xxxxxxxxxx
&a

👉 “a 在哪”

2️⃣ *：解引用（通过地址访问值）

xxxxxxxxxx
*p

👉 “去 p 指向的地方，把值拿出来”

对照理解（记死）

xxxxxxxxxx
p = &a;   // p 记住 a 在哪
*p = 20;  // 把 a 改成 20

三、指针的声明规则（别再乱写）

xxxxxxxxxx
int *p;

❌ 错误理解：*p 是指针 ✅ 正确理解：p 是指针，* 属于 p

易错例子（考试常考）

xxxxxxxxxx
int* p, q;

👉 只有 p 是指针，q 是 int

✔ 推荐写法：

xxxxxxxxxx
int *p;
int q;

四、通过指针修改变量（第一次“真·理解函数”）

错误理解（值传递）

xxxxxxxxxx
void f(int x) {
    x = 10;
}

正确方式（指针）

xxxxxxxxxx
void f(int *x) {
    *x = 10;
}


int main() {
    int a = 5;
    f(&a);
}

📌 函数只能改你给它的地址

五、指针与数组（这一节非常关键）

1️⃣ 数组名就是指针（90% 真）

xxxxxxxxxx
int a[5];

📌 a ≈ &a[0]

2️⃣ 指针访问数组

xxxxxxxxxx
int *p = a;


p[0] == a[0]
*(p + 1) == a[1]

📌 数组访问本质就是指针运算

六、指针运算（很多人怕，其实很简单）

xxxxxxxxxx
int a[5];
int *p = a;

📌 自动按类型跳字节

七、指针 vs Java 引用（帮你彻底理解）

👉 Java 把指针“阉割”成安全引用

八、空指针（NULL）——防崩溃第一守则

xxxxxxxxxx
int *p = NULL;

📌 表示“不指向任何有效内存”

xxxxxxxxxx
if (p != NULL) {
    *p = 10;
}

❌ 解引用 NULL → 程序直接崩

九、指针常见致命错误（请重点背）

1️⃣ 野指针

xxxxxxxxxx
int *p;
*p = 10;   // ❌

2️⃣ 悬空指针

xxxxxxxxxx

    int a = 10;
    p = &a;
}
// a 已销毁

3️⃣ 返回局部变量地址

xxxxxxxxxx
int* f() {
    int x = 10;
    return &x;   // ❌
}

十、二级指针（先感受，不强记）

xxxxxxxxxx

int *p = &a;
int **pp = &p;

四、字符串输入（最容易炸的地方）

第 8 章：结构体（struct）

一、为什么需要结构体？

在 C 语言里，数组只能存同一种类型的数据：

xxxxxxxxxx
int a[10];

但现实世界里的对象，通常是多种属性的组合：

学生 = 学号(int) + 姓名(char[]) + 成绩(float)

如果不用结构体，你只能这样写：

xxxxxxxxxx
int id;
char name[20];
float score;

问题是：

  • 这些变量逻辑上是一组

  • 但代码层面是分散的

  • 传参、管理、扩展都非常痛苦

👉 结构体的作用：把不同类型的数据打包成一个整体

二、结构体的基本定义

1️⃣ 最基本的结构体定义

xxxxxxxxxx
struct Student {
    int id;
    char name[20];
    float score;
};

解释：

  • struct：关键字

  • Student：结构体类型名

  • {} 里是成员变量

⚠️ 注意：

  • 定义结构体 ≠ 定义变量

  • 上面这段代码只是定义了一个“类型”

三、结构体变量的声明

1️⃣ 先定义结构体，再声明变量（最推荐）

xxxxxxxxxx
struct Student stu1;

访问成员用 . 运算符：

xxxxxxxxxx
stu1.id = 1001;
stu1.score = 95.5;

2️⃣ 定义结构体时顺便声明变量

xxxxxxxxxx
struct Student {
    int id;
    char name[20];
    float score;
} stu1, stu2;

不推荐原因：可读性差，维护困难

3️⃣ 使用 typedef（工程中最常见）

xxxxxxxxxx
typedef struct {
    int id;
    char name[20];
    float score;
} Student;

之后直接：

xxxxxxxxxx
Student stu1;

✅ 企业级写法，强烈推荐你以后用这个

四、结构体成员的赋值与访问

1️⃣ 普通成员（int / float）

xxxxxxxxxx
stu1.id = 1001;
stu1.score = 88.5;

2️⃣ char 数组成员（⚠️ 不能直接赋值）

❌ 错误写法：

xxxxxxxxxx
stu1.name = "Tom";   // 错误！

✅ 正确写法：

xxxxxxxxxx
strcpy(stu1.name, "Tom");

📌 原因：

  • 数组名是常量

  • 不能整体赋值

  • 只能拷贝内容

五、结构体的初始化（重点）

1️⃣ 按顺序初始化

xxxxxxxxxx
struct Student stu1 = {1001, "Tom", 90.5};

顺序 必须和定义顺序一致

2️⃣ 指定成员初始化（C99）

xxxxxxxxxx
struct Student stu2 = {
    .name = "Jerry",
    .id = 1002,
    .score = 92.0
};

✅ 好处：

  • 顺序无所谓

  • 不容易写错

六、结构体数组（非常常用）

1️⃣ 定义结构体数组

xxxxxxxxxx
struct Student stu[3];

等价于：

xxxxxxxxxx
3 个 Student 结构体变量

2️⃣ 使用结构体数组

xxxxxxxxxx
stu[0].id = 1001;
strcpy(stu[0].name, "Tom");
stu[0].score = 90;

3️⃣ 结构体数组初始化

xxxxxxxxxx
struct Student stu[2] = {
    {1001, "Tom", 90},
    {1002, "Jerry", 95}
};

七、结构体作为函数参数（关键）

1️⃣ 值传递（会拷贝整个结构体）

xxxxxxxxxx
void printStudent(struct Student s) {
    printf("%d %.1f\n", s.id, s.score);
}

❗ 缺点：

  • 结构体大时，性能差

  • 修改不会影响原变量

2️⃣ 指针传递（强烈推荐）

xxxxxxxxxx
void printStudent(struct Student *s) {
    printf("%d %.1f\n", s->id, s->score);
}

调用：

xxxxxxxxxx
printStudent(&stu1);

📌 -> 是 指针访问结构体成员运算符

八、. 和 -> 的本质区别（必考）

等价关系：

xxxxxxxxxx
s->id
(*s).id

九、结构体与指针的关系（为第 9 章铺路）

xxxxxxxxxx
struct Student stu;
struct Student *p = &stu;

内存结构：

xxxxxxxxxx
p ───▶ stu
         ├── id

         └── score

十、结构体的常见易错点（重点总结）

❌ 1. 忘记 struct 关键字

xxxxxxxxxx
Student stu;   // 错（没有 typedef）

✅ 正确：

xxxxxxxxxx
struct Student stu;

❌ 2. char 数组直接赋值

xxxxxxxxxx

❌ 3. 指针用 . 访问成员

xxxxxxxxxx
p.id   // 错

❌ 4. 未初始化就使用

xxxxxxxxxx
struct Student s;
printf("%d", s.id);  // 野值
