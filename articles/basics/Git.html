3.3 基础操作指令

在 Git 的工作目录中，对文件的修改会经历一系列状态变化。这些状态反映了文件从修改到提交过程中所处的不同阶段，是 Git 工作流的重要组成部分。

1. 🌱 Git 中文件的三大区域（核心概念）

  1. 工作区（Working Directory）：你当前看到和操作的项目目录。

  2. 暂存区（Staging Area / Index）：用来临时保存你打算提交的更改。

  3. 本地仓库（Repository / HEAD）：已提交的历史版本，Git 真正保存快照的地方。

3.4 分支

1. 分支（Branch）是 Git 中非常核心的功能，它允许你在开发中同时进行多个任务，而不会互相干扰。

📌 类比理解：

可以把 Git 分支想象成“故事线”： 你从主线（main）分出来一个副线（dev），在副线中可以自由改动代码，不会影响主线。等副线开发完了，再合并回主线。

2. 指令

  1. git branch :查看分支

  2. git branch +名字：创建分支

  3. git checkout+分支名：切换分支，可以转到一个不存在的分支上，直接创建

  4. git merge +分支名：合并分支

  5. git branch -d/D +分支名：删除分支,D强制删除

3. 查看关系

  1. git-log 查看不同分支的分部和关系

4. 冲突

  1. 在 Git 里，冲突（Conflict）就是：

    Git 在合并（merge）、变基（rebase）、cherry-pick 等操作时，遇到了同一文件的同一位置被两个分支都修改，而且修改内容不一样，Git 无法自动决定用哪一个版本，只能让你人工选择。

    可以理解为：

    • Git 能自动合并不同文件的改动。

    • Git 能自动合并同一文件不同位置的改动。

    • 但如果是同一位置的改动，它懵了 → 冲突。

  2. 合并时面对同一文件的不同操作，会产生

    xxxxxxxxxx
    $ git merge sss
    Auto-merging file1.txt
    CONFLICT (content): Merge conflict in file1.txt
    Automatic merge failed; fix conflicts and then commit the result.

    1. 解决冲突的方法

      1. 打开冲突文件，找到 <<<<<<<、=======、>>>>>>> 标记。

      2. 根据实际需求：

        • 选一个版本

        • 或者手动合并两个版本

      3. 删除冲突标记，保存文件。

      4. 告诉 Git 冲突已解决：

      xxxxxxxxxx
      git add .

      1. 完成合并：

      xxxxxxxxxx
      复制编辑
      git commit

      如果是 merge 过程，这次提交就是“合并提交” 如果是 rebase 过程，继续 git rebase --continue

5. 快进模式

在 Git 里，快进模式（Fast-forward merge） 其实就是一种特殊的合并方式，意思是：

如果目标分支（比如 main）从公共祖先到现在没有新的提交，那么合并时 Git 不需要生成一个新的合并提交，只需要把分支指针“快进”到另一个分支的位置就行。

1. 原理

举个例子：

xxxxxxxxxx
A --- B   (main)
       \
        C --- D   (feature)

这里：

  • main 的最新提交是 B

  • feature 分支从 B 往后多了 C 和 D

  • main 自从 B 之后没有新提交

如果你执行：

xxxxxxxxxx
git switch main
git merge feature

Git 发现：

  • main 的 HEAD 和 feature 的起点是同一个提交 B

  • 所以直接把 main 的指针移动到 D 就行，不需要额外生成一个“合并提交”。

结果：

xxxxxxxxxx
A --- B --- C --- D   (main, feature)
2. 特点

  • 没有新的 merge commit，历史是一条直线，看起来干净。

  • 速度快，所以叫“快进”。

  • 前提：目标分支自公共祖先以来没有额外提交。

3. 如何控制快进模式

  • 允许快进（默认）：

xxxxxxxxxx
git merge feature

  • 禁用快进（强制保留合并提交）：

xxxxxxxxxx
git merge --no-ff feature

这样即使可以快进，也会生成一个 merge commit，保留分支合并的痕迹：

xxxxxxxxxx
A --- B --------- E   (main)
       \       /
        C --- D   (feature)

4.Git远程仓库

1.常用托管服务[远程仓库]

  1. GitHub https://github.com

  2. 码云 https://gitee.com

  3. GitLab https://gitlab.com

2.码云使用

  1. 点开网址注册

  2. 新建仓库

  3. 配置SSH公钥

    1. 在网站安全设置里配置

    2. 打开gitbash 输入ssh-keygen -t rsa，然后回车

    3. 输入 cat ~/.ssh/id_rsa.pub,获取密钥

    4. 验证是否配置成功:ssh-T git@gitee.com

  4. 上传仓库

    1. 打开仓库，复制ssl链接

    2. 打开gitbash输入git remote add origin git@gitee.com:zhang-yi-q/mytest.git:添加远程仓库，固定名origin

    3. git remote:查看远程仓库

    4. git push [-f] [--set-upstream] [远端名称]：传送文件到远程仓库

    5. $ git push --set-upstream origin main:main 可以简化指令

  5. 查看指令

    1. git branch -vv,查看分支与远程的关联

  6. 拉取

    1. 克隆远端仓库，进行更新git clone+复制的ssh地址+名字：最好新建一个

    2. git fetch origin：抓取：将仓库里更新抓取本地

    3. git pull origin :抓取加合并

      1. 解决冲突：打开文件，删除修改

  7. 代码演示

    xxxxxxxxxx
    1.添加远程仓库
    git remote add origin git@网站
    2.将main分支推送到远程仓库，与远程仓库的main绑定
    git push --set-upstream origin main
    3.克隆仓库
    git clone +仓库ssl链接
    4.本地修改提交远程
    git push origin main
    5.git pull 拉去仓库更新
