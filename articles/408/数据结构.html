线性表是具有相同特性的数据元素的一个有限序列。常见的存储结构包括 顺序存储（顺序表） 和 链式存储（链表）。

一、 顺序表 (Sequential List)

顺序表是用一段物理地址连续的存储单元依次存储数据元素的线性结构。

1. 结构定义与初始化

顺序表支持随机访问，查找效率高。

C++

​x
#include <stdio.h>
#include <stdlib.h>


#define MaxSize 10


typedef struct {
    int* data;      // 指向动态分配数组的指针
    int length;     // 当前长度
    int maxsize;    // 最大容量
} SqList;


// 1. 初始化
int InitList(SqList *L) {
    L->data = (int*)malloc(sizeof(int) * MaxSize);
    if (!L->data) return 0;
    L->length = 0;
    L->maxsize = MaxSize;
    return 1;
}

2. 核心操作

  • 插入 (O(n)): 需将目标位置后的元素全部后移。

  • 删除 (O(n)): 需将目标位置后的元素全部前移。

  • 扩容: 申请更大的内存，将原数据拷贝过去。

C++

xxxxxxxxxx
// 2. 插入：在第 i 个位置(1-indexed)插入元素 e
int ListInsert(SqList *L, int i, int e) {
    if (i < 1 || i > L->length + 1) return 0; // 判断范围
    if (L->length >= L->maxsize) return 0;    // 空间已满


    for (int j = L->length; j >= i; j--) {    // 元素后移
        L->data[j] = L->data[j - 1];
    }
    L->data[i - 1] = e;
    L->length++;
    return 1;
}


// 3. 删除：删除第 i 个位置的元素
int ListDelete(SqList *L, int i) {
    if (i < 1 || i > L->length) return 0;


    for (int j = i; j < L->length; j++) {     // 元素前移
        L->data[j - 1] = L->data[j];
    }
    L->length--;
    return 1;
}


// 4. 扩容：动态增加长度
int ExtendList(SqList *L) {
    int *temp = (int*)malloc(sizeof(int) * 2 * L->maxsize);
    if (!temp) return 0;


    for (int i = 0; i < L->length; i++) {     // 数据搬迁
        temp[i] = L->data[i];
    }
    free(L->data);                            // 释放原空间
    L->data = temp;
    L->maxsize *= 2;
    return 1;
}

二、 单链表 (Singly Linked List)

链表通过指针将一组零散的内存块串联起来。插入和删除时不需要移动元素。

1. 结构定义

C++

xxxxxxxxxx
typedef struct LNode {
    int data;           // 数据域
    struct LNode *next; // 指针域
} LNode, *LinkList;

2. 初始化 (带头结点)

提示：带头结点可以简化插入和删除第一个节点时的逻辑。

C++

xxxxxxxxxx
int InitLinkList(LinkList *L) {
    *L = (LNode*)malloc(sizeof(LNode));
    if (*L == NULL) return 0;
    (*L)->next = NULL;
    return 1;
}

3. 链表的建立

  • 头插法：新节点总是在头结点之后插入，结果是逆序的。

  • 尾插法：维护一个尾指针 r，结果是正序的。

C++

xxxxxxxxxx
// 头插法 (Head Insertion)
void CreateListHead(LinkList &L, int n) {
    L = new LNode;
    L->next = NULL;
    for (int i = 0; i < n; i++) {
        LNode* s = new LNode;
        scanf("%d", &s->data);
        s->next = L->next;
        L->next = s;
    }
}


// 尾插法 (Tail Insertion)
void CreateListTail(LinkList &L, int n) {
    L = new LNode;
    LNode *r = L; // r 为尾指针
    for (int i = 0; i < n; i++) {
        LNode* s = new LNode;
        scanf("%d", &s->data);
        s->next = NULL;
        r->next = s;
        r = s;    // 更新尾指针
    }
}

4. 插入与删除 (O(n))

虽然插入本身是 O(1) 的指针操作，但找到第 i 个位置需要 O(n)。

C++

xxxxxxxxxx
// 在第 i 个位置插入 e
int LinkInsert(LinkList L, int i, int e) {
    LNode *p = L;
    int j = 0;
    while (p && j < i - 1) { // 寻找第 i-1 个结点
        p = p->next;
        j++;
    }
    if (!p) return 0;


    LNode *s = (LNode*)malloc(sizeof(LNode));
    s->data = e;
    s->next = p->next;
    p->next = s;
    return 1;
}


// 删除第 i 个位置元素
int LinkDelete(LinkList L, int i) {
    LNode *p = L;
    int j = 0;
    while (p && j < i - 1) {
        p = p->next;
        j++;
    }
    if (!p || !p->next) return 0;


    LNode *q = p->next;     // 暂存待删节点
    p->next = q->next;      // 断链
    free(q);                // 释放空间
    return 1;
}

折半查找

折半查找（Binary Search）： 在 有序表 中，每次把查找区间 一分为二，通过比较中间元素与目标值，缩小查找范围。

必须满足：

  1. 顺序存储结构（数组）

  2. 关键字有序（递增或递减

❌ 以下情况不能直接用折半查找：

  • 链表（无法 O(1) 访问中间）

  • 无序数组

xxxxxxxxxx
#include <iostream>
#include <vector>
using namespace std;


int binarySearch(vector<int>& nums, int target) {
    int low = 0;
    int high = nums.size() - 1;
    
    while (low <= high) {                 // ✅ 1. 条件修正
        int mid = (low + high) / 2;       // ✅ 2. mid 放在循环里
        
        if (nums[mid] == target) {
            return mid;
        }
        else if (nums[mid] > target) {
            high = mid - 1;               // ✅ 3. 左半区
        }
        else {
            low = mid + 1;                // ✅ 4. 右半区
        }
    }
    
    return -1;
}




int main() {
    vector<int> nums = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};
    int target = 23;
    
    int index = binarySearch(nums, target);
    
    cout << index << endl;
    return 0;
}

 

折半查找的判定树

一、什么是折半查找的判定树（概念）

判定树：

描述折半查找过程中，比较关键字先后顺序的一棵二叉树。

换句话说：

  • 每一次和 A[mid] 的比较

  • 都对应判定树中的 一个结点

  • 从根走到某个结点 = 一次完整的查找过程

📌 关键点（必背）：

  • 判定树是 二叉树

  • 用来分析 查找长度（ASL）

  • 是“查找过程”的抽象，而不是数据结构本身

二、折半查找判定树怎么画（老师最爱让你画） 示例：有序表

xxxxxxxxxx
{2, 5, 8, 12, 16, 23, 38}

元素个数 n = 7

第一步：找中间元素

xxxxxxxxxx
mid = (0 + 6) / 2 = 3
→ 根结点 = 12

第二步：左右区间继续折半

左半区 {2, 5, 8} → 中间：5

右半区 {16, 23, 38} → 中间：23

第三步：继续直到区间为空

得到判定树：

xxxxxxxxxx
          12
        /    \
       5      23
      / \    /  \
     2   8  16  38

⚠️ 注意：

  • 这是判定树，不是二叉排序树

  • 虽然“看起来像”，但概念不同（下面说）

     

判定树的性质

  1. 性质 1️⃣：判定树是一棵「近似完全二叉树」

    原因：

    • 每次查找区间都 对半分

    • 上层一定是满的

    • 最后一层可能不满

    📌 结论（直接写）：

    折半查找的判定树是一棵近似完全二叉树。

    性质 2️⃣：判定树的高度是 ⌈log₂(n+1)⌉

    也常写成：

    xxxxxxxxxx
    h ≈ log₂ n

    👉 这直接导出时间复杂度：

    折半查找时间复杂度为 O(log₂ n)

    性质 3️⃣：判定树 ≠ 二叉排序树（超高频陷阱⚠️）

    虽然：

    • 左子树 < 根 < 右子树（数值上）

    但：

    性质 4️⃣：成功查找结点 = 有序表元素

    • 每一个表中元素

    • 都对应判定树中的一个结点

    • 成功查找次数 = 到该结点的路径长度

    性质 5️⃣：失败查找结点在“外部结点”

    • 判定树的空指针位置

    • 表示“查找失败”

    • 失败查找次数与这些外部结点有关

    📌 结论（会写就赢）：

    折半查找的失败结点数比成功结点数多 1

    （即：n + 1 个失败位置）

七、408 最爱考的问法 & 标准答句

❓ 问：

为什么折半查找的失败结点数是 n+1？

✅ 标准回答（你可以直接用）：

在折半查找的判定树中，成功查找对应内部结点，失败查找对应外部结点。 根据二叉树性质，外部结点数等于内部结点数加一，因此含 n 个元素的有序表，其失败查找结点数为 n+1。

 

ASL

1️⃣ 查找长度是什么？

从根结点到目标结点的比较次数

也就是：

  • 判定树中结点的层数

2️⃣ 成功查找 ASL

公式思想（不一定要求写公式）：

xxxxxxxxxx
ASL成功 = 各成功结点比较次数之和 / 成功结点数

由于判定树接近完全二叉树：

ASL成功 ≈ log₂ n

3️⃣ 失败查找 ASL

  • 走到“空指针”

  • 比成功略大

  • 但仍是 O(log₂ n)
