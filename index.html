2.2 关系代数

关系代数是一种抽象的查询语言，以关系为运算对象，通过一组形式化的运算符来描述对数据库中数据的检索与处理过程。 关系代数运算主要分为 集合运算 和 关系运算 两大类。

1. 集合运算

集合运算是以关系作为集合进行操作的运算，要求参与运算的关系通常具有相同的属性结构。

常见的集合运算包括：

  1. 并（∪）： 将两个关系中的元组合并，结果中包含属于任一关系的元组。

  2. 交（∩）： 结果关系中仅包含同时属于两个关系的元组。

  3. 差（−）： 从第一个关系中去掉也出现在第二个关系中的元组。

  4. 补： 在给定全集的前提下，取某关系在全集中的补集。

  5. 笛卡尔积（×）： 将两个关系中的每个元组两两组合，生成一个新的关系，是后续连接运算的基础。

以上均属于二目运算，即需要两个关系作为操作对象。（教材 P30）

示例： 设关系 R(A,B) 与 S(A,B)

  • R ∪ S 表示 R 与 S 的并集

  • R − S 表示在 R 中但不在 S 中的元组

2. 关系运算

关系运算是关系代数中的核心部分，用于对关系进行更灵活、更精确的操作。在关系运算中，引入了以下基本概念和符号。

（1）分量

分量是指元组中某个属性对应的具体取值。

示例： 若有元组

xxxxxxxxxx
t = (1001, 张三, 20)

其中：

  • 1001、张三、20 都是分量

（2）属性组

属性组是指关系中若干属性的集合，通常用于投影或连接运算。

示例： 关系 Student(Sno, Name, Age, Dept)

  • {Sno, Name} 是一个属性组

（3）元组的连接

元组连接是指将来自不同关系的元组按照一定条件进行组合，生成新的元组。 这是连接运算（⋈）的基础。

示例：

  • 学生表 Student(Sno, Name, DeptId)

  • 系表 Dept(DeptId, DeptName)

当 Student.DeptId = Dept.DeptId 时，可将两个元组连接成一个新元组，包含学生和系的信息。

（4）象集

象集是指：

对关系中的某个元组，在指定属性组上的取值集合。

通俗理解：从一个元组中“取出”某些属性形成的新集合。

示例： 关系 Student(Sno, Name, Age, Dept) 元组：

xxxxxxxxxx
t = (1001, 张三, 20, CS)

在属性组 {Name, Dept} 上，元组 t 的象集为：

xxxxxxxxxx
{张三, CS}

第六章 数据查询

1. SELECT 语句

SELECT 语句用于从数据库中查询数据，是最常用的 SQL 语句之一。

  1. SELECT 子句：指定要显示的列 用于指定查询结果中显示的字段。

    示例：

    xxxxxxxxxx
    SELECT Sno, Name FROM Student;

  2. FROM 子句：指定查询数据来源的表或视图 指定要查询的数据表。

    示例：

    xxxxxxxxxx
    SELECT * FROM Student;

  3. WHERE 子句：选择行的条件 用于限定查询的条件。

    示例：

    xxxxxxxxxx
    SELECT * FROM Student
    WHERE Age > 20;

  4. GROUP BY 子句：指定分组表达式 将查询结果按指定字段进行分组，通常与聚合函数一起使用。

    示例：

    xxxxxxxxxx
    SELECT Age, COUNT(*)
    FROM Student
    GROUP BY Age;

  5. HAVING 子句：指定满足分组的条件 对分组后的结果进行筛选。

    示例：

    xxxxxxxxxx
    SELECT Age, COUNT(*)
    FROM Student
    GROUP BY Age
    HAVING COUNT(*) > 1;

  6. ORDER BY 子句：指定行的排序 对查询结果进行升序或降序排列。

    示例：

    xxxxxxxxxx
    SELECT * FROM Student
    ORDER BY Age DESC;

  7. LIMIT 子句：指定查询结果集包含的行数 用于限制查询返回的记录条数。

    示例：

    xxxxxxxxxx
    SELECT * FROM Student
    LIMIT 5;

  1. 投影指定的列 只显示表中指定的字段。

  2. 投影全部的列 使用 * 显示表中的所有字段。

    示例：

    xxxxxxxxxx

  4. 计算列值 在查询中对字段进行运算，生成新的结果列。

    示例：

    xxxxxxxxxx
    SELECT Name, Age + 1 AS NextAge FROM Student;

  5. 去掉重复行 使用 DISTINCT 关键字去除查询结果中的重复记录。

    示例：

    xxxxxxxxxx
    SELECT DISTINCT Age FROM Student;

3. 选择查询

    示例：

    xxxxxxxxxx

  4. 使用 LIKE 进行匹配查询

    1. %：匹配任意长度的字符串

      示例：

      xxxxxxxxxx
      SELECT * FROM Student
      WHERE Name LIKE '张%';

    2. _：匹配单个字符

      示例：

      xxxxxxxxxx
      SELECT * FROM Student

  5. 使用正则表达式进行查询 使用 REGEXP 或 RLIKE 进行正则匹配。

    示例：

    xxxxxxxxxx
    SELECT * FROM Student
    WHERE Name REGEXP '^张';

4. 分组查询和统计计算

分组查询用于对查询结果进行分组，并进行统计计算。

  1. 聚合函数

    1. COUNT：统计记录数

      xxxxxxxxxx
      SELECT COUNT(*) FROM Student;

    2. SUM：计算总和

      xxxxxxxxxx
      SELECT SUM(Age) FROM Student;

    3. AVG：计算平均值

      xxxxxxxxxx
      SELECT AVG(Age) FROM Student;

    4. MAX：求最大值

      xxxxxxxxxx
      SELECT MAX(Age) FROM Student;

  2. GROUP BY 子句 按指定字段进行分组。

    示例：

    xxxxxxxxxx
    SELECT Age, COUNT(*)
    FROM Student
    GROUP BY Age;

  3. HAVING 子句 对分组后的结果进行条件筛选。

    示例：

    xxxxxxxxxx

5. 排序查询和限制查询结果的数量

连接查询用于从多个表中获取相关数据。

1. 交叉连接

又叫做笛卡尔积，返回两个表中所有可能的组合。

示例：

xxxxxxxxxx
SELECT * FROM Student CROSS JOIN Course;

2. 内连接

内连接只返回满足连接条件的记录。

  1. 使用 INNER JOIN 的显式语法结构

    示例：

    xxxxxxxxxx
    SELECT *
    FROM Student
    INNER JOIN Course
    ON Student.Cno = Course.Cno;

  2. 使用 WHERE 子句定义的隐式连接

    示例：

    xxxxxxxxxx
    SELECT *
    FROM Student, Course
    WHERE Student.Cno = Course.Cno;

  3. 种类

    1. 等值连接和非等值连接

      xxxxxxxxxx
      SELECT *
      FROM Student s, Score sc
      WHERE s.Sno = sc.Sno;

    2. 自然连接 自动按同名字段连接。

      xxxxxxxxxx
      SELECT * FROM Student NATURAL JOIN Course;

    3. 自连接 表与自身进行连接。

      xxxxxxxxxx
      SELECT a.Name, b.Name
      FROM Student a, Student b
      WHERE a.Class = b.Class;

3. 外连接

外连接在连接结果中保留一侧表的所有记录。

  1. 左外连接

    示例：

    xxxxxxxxxx
    SELECT *
    FROM Student
    LEFT JOIN Score
    ON Student.Sno = Score.Sno;

  2. 右外连接

    示例：

    xxxxxxxxxx
    SELECT *
    FROM Student
    RIGHT JOIN Score
    ON Student.Sno = Score.Sno;

子查询是指在一个查询语句中嵌套另一个查询语句。

  1. IN 子查询

    示例：

    xxxxxxxxxx

      SELECT Sno FROM Score WHERE Grade > 80
    );

  2. 比较子查询

    示例：

    xxxxxxxxxx
    SELECT * FROM Student
    WHERE Age > (
      SELECT AVG(Age) FROM Student
    );

  3. EXISTS 子查询

    示例：

    xxxxxxxxxx
    SELECT * FROM Student s
    WHERE EXISTS (
      SELECT * FROM Score sc
      WHERE s.Sno = sc.Sno
    );

8. 联合查询

联合查询用于将多个 SELECT 语句的查询结果合并为一个结果集。

  • 各 SELECT 语句返回的列数和数据类型必须一致

  • 默认去除重复记录，使用 UNION ALL 保留重复记录

示例：

xxxxxxxxxx
SELECT Name FROM Student
UNION
SELECT Name FROM Teacher;

 

第七章 视图和索引

视图：是从一个或多个表导出的虚拟表，用来导出视图的表称为基表或基本表，导出的视图称为虚表。视图本身不存储数据，数据仍然保存在基表中。

视图的优点

  1. 方便用户查询

  2. 保护数据安全

  3. 便于数据共享

  4. 简化查询操作

  5. 可以重新组织数据

  3. 更新视图

    1. 可更新视图有 基于单表、不包含聚合函数、DISTINCT、GROUP BY 的视图通常是可更新的。

    2. 插入数据

      xxxxxxxxxx
      INSERT INTO v_student VALUES (1005, '周七', 20);

    3. 修改数据

      xxxxxxxxxx
      UPDATE v_student
      SET Age = 21
      WHERE Sno = 1005;

    4. 删除数据

      xxxxxxxxxx
      DELETE FROM v_student
      WHERE Sno = 1005;

  4. 修改视图 使用 CREATE OR REPLACE VIEW 重新定义视图。

    示例：

    xxxxxxxxxx
    CREATE OR REPLACE VIEW v_student
    AS
    SELECT Sno, Name FROM Student;

  5. 删除视图 使用 DROP VIEW 删除视图。

    示例：

    xxxxxxxxxx
    DROP VIEW v_student;

索引是按照数据表中的一列或多列进行排序，并为其建立指向数据表记录所在位置的指针，从而提高数据检索效率。

2. 索引的分类

  1. 普通索引

  2. 唯一性索引

  3. 主键

  4. 聚簇索引

  5. 全文索引

3. 索引的使用

索引可以加快查询和检索速度，但需要增加存储空间，并在对表进行插入、删除和更新操作时，降低更新性能。

4. 索引操作

  1. 创建索引

    1. 使用 CREATE INDEX

      xxxxxxxxxx
      CREATE INDEX idx_name ON Student(Name);

    2. 使用 ALTER TABLE

      xxxxxxxxxx
      ALTER TABLE Student ADD INDEX idx_age(Age);

  2. 查看表上建立的索引

    xxxxxxxxxx
    SHOW INDEX FROM Student;

  3. 删除索引

    1. 使用 DROP INDEX

      xxxxxxxxxx
      DROP INDEX idx_name ON Student;

    2. 使用 ALTER TABLE

      xxxxxxxxxx
      ALTER TABLE Student DROP INDEX idx_age;

第十章 存储过程和存储函数

存储过程是一组完成特定功能的 SQL 语句集，存放在数据库服务器中。 用户只需要调用存储过程名即可执行其中的 SQL，提高执行效率并增强安全性。

  • 可以封装复杂业务逻辑

  • 减少网络传输

  • 提高代码复用性

  • 增强数据库安全性

​x
delimiter $$


create procedure proc_test()
begin
    select * from student;
end $$

第十四章 事务和锁

1. 事务的基本概念

1. 事务的概念

事务是数据库操作的最小执行单元，由一组 SQL 语句组成，这些语句要么全部成功执行，要么全部失败回滚。

2. 事务的特性

事务具有 ACID 四个特性：

  1. 原子性（Atomicity）：事务中的操作不可再分

  2. 一致性（Consistency）：事务执行前后数据保持一致

  3. 隔离性（Isolation）：事务之间互不干扰

  4. 持久性（Durability）：事务提交后永久保存

2. 事务控制语句

1. 开启事务

xxxxxxxxxx

2. 提交事务

xxxxxxxxxx

3. 撤销事务

xxxxxxxxxx

3. 事务的并发处理

1. 三种隔离级别

MySQL 支持多种事务隔离级别：

  1. 读未提交（read uncommitted）

  2. 读已提交（read committed）

  3. 可重复读（repeatable read，默认级别）

xxxxxxxxxx
set transaction isolation level repeatable read;

 

2. 解决四个问题

事务隔离级别用于解决并发访问产生的问题：

1. 脏读

概念： 脏读是指一个事务读取到了另一个事务尚未提交的数据。

产生原因： 一个事务对数据进行了修改，但还未提交，另一个事务就读取了该数据。如果前一个事务回滚，后一个事务读到的数据就是无效的“脏数据”。

示例说明：

  • 事务 A 修改账户余额，但未提交

  • 事务 B 读取了这个未提交的余额

  • 事务 A 回滚 ➡ 事务 B 读到的数据是错误的

影响： 会导致读取的数据不真实，严重影响数据一致性。

2. 不可重复读

概念： 不可重复读是指在同一个事务中，对同一条记录进行两次读取，结果却不一致。

产生原因： 在事务执行过程中，另一个事务提交了对该记录的修改。

示例说明：

  • 事务 A 第一次读取某条记录

  • 事务 B 修改该记录并提交

  • 事务 A 再次读取该记录，发现值发生变化 ➡ 两次读取结果不一致

影响： 破坏了事务内部读取结果的一致性。

3. 幻读

概念： 幻读是指在同一个事务中，两次执行相同条件的查询，返回的记录条数发生变化。

产生原因： 在事务执行期间，其他事务插入或删除了符合查询条件的新记录并提交。

示例说明：

  • 事务 A 查询满足条件的记录，共 5 条

  • 事务 B 插入一条符合条件的新记录并提交

  • 事务 A 再次查询，结果变成 6 条 ➡ 多出来的记录称为“幻影记录”

影响： 影响统计查询和范围查询的正确性。

4. 更新丢失

概念： 更新丢失是指多个事务同时更新同一数据，其中一个事务的更新结果被另一个事务覆盖，导致数据修改丢失。

产生原因： 事务之间缺乏必要的并发控制。

示例说明：

  • 事务 A 读取数据并修改

  • 事务 B 也读取相同数据并修改

  • 事务 A 提交

  • 事务 B 提交，覆盖了事务 A 的结果 ➡ 事务 A 的更新被丢失

影响： 直接导致数据不正确，是严重的并发问题。

第十五章 关系数据库设计理论

1. 概述

关系数据库设计理论由 E. F. Codd 提出，是关系数据库系统建立的理论基础。 其核心目标是：减少数据冗余、避免更新异常、提高数据一致性和可维护性。

关系数据库设计理论主要研究以下内容：

  1. 函数依赖 描述关系中属性之间的依赖关系，是规范化理论的基础。

  2. 范式 用于衡量关系模式合理程度的标准，通过规范化消除数据冗余和异常。

  3. 模式设计 在满足需求的前提下，设计结构合理、性能良好的关系模式。

2. 关系模式规范化

规范化是将一个关系模式分解为若干个合理的关系模式的过程。

  1. 1NF 要求关系中的每一个属性都是不可再分的原子值，不存在重复组或多值属性。

  2. 2NF 在满足 1NF 的基础上，消除非主属性对主键的部分函数依赖。

  3. 3NF 在满足 2NF 的基础上，消除非主属性对主键的传递函数依赖。

  4. 多值依赖与4NF 当关系中存在多值依赖时，需要进一步分解以满足 4NF，从而消除多值冗余。

3. 数据依赖的公理系统

1. Armstrong公理系统

Armstrong 公理系统是一组用于推导函数依赖的基本规则，包括：

  • 自反律

  • 增广律

  • 传递律

该公理系统是完备且可靠的，可用于推导所有函数依赖。

2. 闭包及其计算

属性集的闭包是指在给定函数依赖集下，一个属性集能够推导出的所有属性的集合。 闭包常用于判断函数依赖、候选键以及模式分解的合理性。

第十六章 数据库设计

1. 概述

数据库设计是指根据用户需求，建立满足应用要求的数据库结构的全过程。 目标是在保证数据完整性和一致性的前提下，提高系统的可扩展性和运行效率。

2. 需求分析

需求分析是数据库设计的基础阶段，主要任务是：

  • 调查用户信息需求

  • 分析数据内容和处理方式

  • 确定系统的功能需求和性能需求

3. 概念结构设计

概念结构设计是将需求分析结果抽象为概念模型的过程。 通常使用 E-R 模型 表示实体、属性及实体之间的联系。

4. 逻辑结构设计

逻辑结构设计是将概念模型转换为逻辑模型的过程。 主要工作包括：

  • 确定关系模式

  • 选择主键和外键

  • 进行规范化处理

5. 物理结构设计

物理结构设计是根据数据库管理系统的特点，设计数据在物理介质上的存储方式。 包括存储引擎选择、索引设计、存储路径规划等。

6. 数据库实施

数据库实施是将设计结果付诸实现的阶段，主要内容包括：

  • 创建数据库和表

  • 编写应用程序

  • 数据加载和测试

7. 数据库运行和维护

数据库运行和维护是数据库投入使用后的持续工作，包括：

  • 数据备份与恢复

  • 性能监控与调优

  • 权限管理和安全维护

  • 系统升级和故障处理
